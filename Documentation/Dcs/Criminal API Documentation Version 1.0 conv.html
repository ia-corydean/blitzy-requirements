DEVELOPER MANUAL
DCS Criminals Data API – Version 1.0
Introduction –
This documentation provides specifications for DCS’ Criminal Data REST API Service. This is revision 1.0 dated
10/30/2019. The DCS Criminal API provides access to criminal offense information via the criminal’s name/date of birth.
All criminal information is searched based on last name/first name/date of birth because criminal profile data does not
contain a key data such as a social security number, driver’s license number or other key information.
Version Changes
This version is the initial version and leverages logic from the Household Drivers Report Criminal data. At the current
time this is a beta release and access is only on a test/development basis. We expect the production release around
11/6/2019.
System Requirements
These web services are implemented using REST over HTTPS.
Security and Authentication
Given the protected and sensitive nature of data involved in this service, access is both encrypted and restricted. The
encryption of request and response data is protected at the transport layer via the HTTPS protocol. Authentication and
authorization is provided through the use of HTTP Basic Authentication. Using this protocol, an authorization header is
created with Account Number, User ID and Password separated by colons (“:”) and encoded into a Base64 string and
added into the Post request. The account number is your assigned account number from DCS. The customer
determines the username and password for the interactive website but the username and password are assigned for the
API.
Transaction Request
Requests are made via a Post verb. The body of the request contains the CriminalRequest formatted data. URI is
supplied below. The site listed is a test site and will change once production transactions begin. Requests are made
using XML or JSON formatted data and responses are returned also using XML or JSON. This documentation uses XML
examples. If you have questions about JSON requests, please contact us but the XML is simply converted to JSON. The
same data structures are used for both.
